Red Básica

Una red básica conecta dispositivos finales para permitir la comunicación entre ellos. Normalmente incluye un router, un switch y equipos como una computadora y una laptop, y se configura con direcciones IP que pertenecen a la misma subred.

En Cisco Packet Tracer, una red básica permite entender cómo trabaja el Router 2911, cómo el Switch 2960 administra el tráfico y cómo los dispositivos finales intercambian datos.

Conceptos clave:

  • Router: dispositivo que enruta paquetes entre redes diferentes.
  • Switch: dispositivo que conmutar tramas dentro de una misma red local.
  • Subred: segmento de red con un rango de direcciones IP compartidas.

Packet Tracer

Cisco Packet Tracer es un simulador de redes diseñado por Cisco para practicar la creación y configuración de topologías sin necesitar hardware físico. Permite colocar routers, switches, PCs y laptops, aplicar comandos Cisco IOS y comprobar el comportamiento de la red.

Packet Tracer es una herramienta formativa que reproduce conceptos reales de redes, como direcciones IP, máscaras de subred, protocolos de enrutamiento y VLANs.

¿Qué se puede hacer con Packet Tracer?

  • Crear topologías completas con routers, switches, PCs, laptops, servidores y otros dispositivos.
  • Configurar interfaces, direcciones IP, VLANs, enrutamiento estático y dinámico, y seguridad básica.
  • Simular tráfico entre dispositivos y comprobar la comunicación con comandos como ping y traceroute.
  • Practicar el uso de comandos Cisco IOS, acceso remoto y administración de redes.
  • Guardar proyectos y regresar a ellos para seguir aprendiendo o mejorando la configuración.

Cómo Crear la Red

Para crear una red básica debes combinar diseño y configuración. Primero se define la topología física, luego se eligen los equipos y finalmente se asignan las direcciones IP y las conexiones necesarias para que los dispositivos se comuniquen.

Conceptos clave:

  • Topología física: disposición de los dispositivos y cables en el área de trabajo.
  • Tipos de cable: cobre directo para conectar PCs y switch, y cable cruzado cuando se conecta un router a otro router o equipo similar.
  • Puerta de enlace: dirección IP del router que permite a los hosts salir de su subred.

Router 2911

El Router 2911 actúa como el punto de salida de la red local y la puerta de enlace para los hosts. Su configuración inicial implica habilitar interfaces y asignar direcciones IP.

Conceptos clave:

  • Interfaz: puerto del router donde se conecta un cable y se configura una dirección IP.
  • IP de gateway: dirección utilizada por los hosts para enviar tráfico fuera de su subred.
  • CLI: línea de comandos de Cisco IOS usada para configurar el router.

Ejemplo de comandos básicos en la CLI:

enable
configure terminal
interface GigabitEthernet0/0
ip address 192.168.1.1 255.255.255.0
no shutdown
exit

Switch 2960

El Switch 2960 actúa como el centro de conmutación de la LAN. Su función principal es recibir tramas y reenviarlas al puerto correcto según la dirección MAC de destino.

Conceptos clave:

  • Tabla MAC: lista que el switch construye para saber en qué puerto está cada dirección MAC.
  • VLAN: red virtual que permite segmentar el tráfico dentro de un mismo switch.
  • Dominio de colisión: área de la red donde los paquetes pueden chocar, reducida por el uso de switches.

PC y Laptop

La PC y la laptop son los dispositivos finales de la red. Para comunicarse correctamente, cada uno debe tener una dirección IP única en la misma subred y apuntar a la puerta de enlace correcta.

Conceptos clave:

  • Dirección IP: identificador lógico que permite ubicar el dispositivo en la red.
  • Máscara de subred: define qué parte de la IP corresponde a la red y qué parte a los hosts.
  • Puerta de enlace predeterminada: dirección IP del router que se usa para salir de la red local.
